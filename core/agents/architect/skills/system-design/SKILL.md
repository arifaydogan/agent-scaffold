# System Design

1. List requirements, constraints, and unknowns.
2. Draw the data flow and ownership boundaries.
3. Prefer one deployable unit until scaling evidence requires separation.
4. Record failure modes, observability, and migration cost.
5. Produce an ADR for decisions that are expensive to reverse.
