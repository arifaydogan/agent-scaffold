# Antigravity Model Routing

Follow the provider-neutral model and token-routing policy in the repository
root `PACEBUILD_ORCHESTRATOR.md`.

When Antigravity exposes a model or reasoning selector, use its lowest-cost
capable option for Jira/Confluence intake, documentation, child ordering, and
ordinary review. Escalate only for ambiguous cross-service work, repeated
verification failures, security, concurrency, migrations, authorization, or
data-loss risk.

For an independent review, start one fresh read-only agent with the Jira key or
pull-request URL. It must read the Jira review packet and linked Confluence
validation page before reviewing the branch. Do not run parallel reviewers
unless the user explicitly requests them.
